# /cv-update-heatmap

## STEP 1：対象サイトを確認する

以下から対象サイトを選んでもらう。

```
どのサイトのCV記事を分析しますか？
honbe / gyn / asami / ginza / smc
```

---

## STEP 2：スプシを読み込む

維持記事管理シート（ID：`1yr1mz3nQmk2YW2qkjlP0wqZyzdK5ReECz7TdsD9KJfw`）の `[サイト]-CV` タブを読み込む。

全記事の記事名・URLを取得して一覧を提示する。

---

## STEP 3：スクショを収集・分析する

記事ごとに順番にスクショを依頼する。受け取り次第、即分析する。

```
[記事名]のClarityスクショを貼り付けてください。

① クリックヒートマップ
② スクロールヒートマップ
③ アテンションヒートマップ
```

1記事の分析が完了したら次の記事へ進む。全記事完了まで繰り返す。

---

## STEP 4：分析結果を提示する

全記事の分析結果を一覧で提示する。確認なしに書き込まない。

```
【ヒートマップ分析結果（[サイト名]）】

■ [記事名]
  [YYYY-MM-DD] ①[改善ポイント] ②[改善ポイント] …

■ [記事名2]
  ...
```

---

## STEP 5：P列に書き込む

書込前チェック → [P列: ヒートマップ改善ポイント] | [STEP 3分析結果] | [関連セル矛盾なし?]

承認を得てから全記事分まとめてP列に書き込む（上書き）。

---

## STEP 6：ログをmdに追記する

`logs/[サイト名]-cv-log.md` に以下の形式で追記する。ファイルが存在しない場合は新規作成する。

```markdown
## [YYYY-MM-DD] W3 ヒートマップ分析

### [記事名]
**ヒートマップ改善ポイント：**
① xxx ② xxx …

---
```

---

## STEP 7：Slack通知テキストを出力する

```
[サイト名]のヒートマップ分析が完了しました。
P列を確認して、Q列に採用番号を記入してください。
```
