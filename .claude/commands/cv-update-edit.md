# /cv-update-edit

## STEP 1：対象サイトを確認する

以下から対象サイトを選んでもらう。

```
どのサイトのCV記事を実装しますか？
honbe / gyn / asami / ginza / smc
```

---

## STEP 2：対象記事を取得する

維持記事管理シート（ID：`1yr1mz3nQmk2YW2qkjlP0wqZyzdK5ReECz7TdsD9KJfw`）の `[サイト]-CV` タブを読み込む。

**対象条件**：P列（社員対応番号）が入力済みかつM列（更新完了日）が空

以下の形式でインターンに提示する。確認を得てから次へ進む。

```
実装待ちの記事

記事名：[記事名]
URL：[URL]
採用番号：[P列の番号]

この記事を進めましょう。
```

---

## STEP 3：採用された方針を確認する

P列の番号をもとにN列またはO列から該当方針を抽出して表示する。

```
採用された改修方針：
[番号] [方針内容]

この方針で改修を進めます。
```

---

## STEP 4：HTMLを受け取る

```
対象記事のHTMLをWordPressの「テキスト」タブから全文コピーして貼り付けてください。
```

受け取ったHTMLのタグ構造・class名・文体を読み取る。

実施前に以下を読む。
- `knowledge/block-rules.md`
- `knowledge/guidelines.md`（2章：見出し構成のみ）

---

## STEP 5：HTML差分を出力する

採用方針に沿った差分HTMLを出力する。
元のHTMLのタグ構造・class名・文体に合わせる。
WPの「テキスト」タブに貼り付けて実装できる状態で出す。

---

## STEP 6：スプシを更新する（WP実装完了後）

WP実装が完了したら報告してもらう。報告を受けてからM列に書き込む。

書込前チェック → [M列: 更新完了日] | [今日の日付（YYYY-MM-DD）] | [関連セル矛盾なし?]

承認を得てからM列に書き込む。
