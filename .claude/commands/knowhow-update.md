# ノウハウ記事 定期更新フロー

## 概要

| | W1 | W2 |
|--|--|--|
| 頻度 | 月1回目 | 月2回目 |
| ネタ | 競合分析 | SERPチェック |
| 起動者 | インターン | インターン |

記事数が多いサイトは更新完了日をもとに半分ずつ更新する（グループ列なし・Claude が自動判定）。
記事数が少ないサイトは全件対象。

---

## 起動方法

Claude Codeで `/knowhow-update` を起動し、以下を伝える。

```
対象タブ：[サイト名]-ノウハウ
今日の日付：[YYYY-MM-DD]
今月の実施回数：1回目 or 2回目
```

---

## Claudeの実行フロー

### STEP 1：対象記事を特定する

MCPで維持記事管理シート（ID：`1yr1mz3nQmk2YW2qkjlP0wqZyzdK5ReECz7TdsD9KJfw`）の対象タブを読み込む。

以下のルールで今回の対象記事を絞り込む。

1. **対象記事の判定**
   - K列（更新完了日）を確認する
   - 更新完了日が入っているサイトで記事数が多い場合 → 更新完了日が古い順に半分を今回の対象とする
   - 更新完了日が空欄（未更新）の記事が多い場合 → 未更新のものを優先して半分を対象とする
   - 記事数が少ないサイト（目安：5件以下）→ 全件対象

2. **W1 or W2の判定**
   - 今月の実施回数が1回目 → W1（競合分析）
   - 今月の実施回数が2回目 → W2（SERPチェック）

3. **空欄スルー**
   - 必要列（W1ならI・J・K列、W2ならF・G・H列）が空欄の記事はスキップする

対象記事の一覧と、インターンに記入を求める情報をまとめて出力する。確認なしに次へ進まない。

---

### STEP 2：インターンへの記入指示を出力する

対象記事ごとに、スプシのどの列に何を記入すればよいかを出力する。

**W1のとき：**
```
以下の記事について、スプシの対象行に競合構成①②③を記入してください。

・[記事名]
  競合構成①（I列）：メインKW「○○」で検索したとき1位の記事のH2/H3見出しをすべて記入
  競合構成②（J列）：2位の記事のH2/H3見出し
  競合構成③（K列）：3位の記事のH2/H3見出し

記入が完了したら「完了しました」と返信してください。
```

**W2のとき：**
```
以下の記事について、スプシの対象行にSERPデータを記入してください。

・[記事名]
  AIO（F列）：メインKW「○○」でGoogle検索したときのAI概要全文（表示なしなら「表示なし」）
  PAA（G列）：PAA上位3つをセル内改行（Ctrl+Enter）で区切って記入
  関連KW（H列）：検索結果最下部の「関連する検索」全て

記入が完了したら「完了しました」と返信してください。
```

インターンの「完了しました」を受けてから次へ進む。

---

### STEP 3：スプシを再読み込みして候補を出力する

インターンの記入完了を受けて、スプシを再度読み込む。

記事ごとに「自社記事に不足しているトピックの候補」をリストアップする。

**W1のとき：**
競合3記事のH2/H3と自社記事を比較する。
判定基準：記事本文でそのトピックに実質的に回答されているか。H3がなくても本文で回答済みなら候補にしない。

**W2のとき：**
AIO・PAA・関連KWを確認し、記事本文で未回答のトピックを候補にする。
前回更新完了日から変化したものを優先的に出す。

候補を出力したらインターンの返答を待つ。確認なしに次へ進まない。

---

### STEP 4：入れ方・入れる場所を提案する

インターンが採用するトピックを返信したら、トピックごとに提案する。

- 既存のどのH3に・どこに組み込むか
- または新規H3として追加するか（どのH2の下・どの位置に挿入するか）

提案を出力したらインターンの返答を待つ。確認なしに次へ進まない。

---

### STEP 5：HTMLを出力する

インターンが承認・修正の返答をしたら、HTMLを出力する。

実施前に以下を読む。
- `knowledge/block-rules.md`
- `knowledge/guidelines.md`（2章：見出し構成 のみ）

元記事の文体（ですます調・表現・H3の書き方）を投入された記事全文から読み取り、スタイルを統一する。
WPに貼り付けて実装できる状態で出す。

---

### STEP 6：更新完了日をスプシに書き込む（記事ごとに即時実行）

WP実装が完了した記事ごとに、その場でMCPでL列（更新完了日）に今日の日付（YYYY-MM-DD）を書き込む。
全記事が終わってからまとめて書き込まない。途中でセッションが切れても完了済み記事の記録が残るようにするため。

次の記事に進む前に必ずL列への書き込みを完了させる。

---

### STEP 7：記録フォーマットを出力する

以下を出力する。インターンはコピペして記録シートに手動で貼る。

```
記事名：
メインKW：
実施日：
パターン：W1 / W2
実施内容：追加 / 強化 / 両方 / 変更なし
変更箇所：
　追加したH3：（該当する場合）
　強化したH3：（該当する場合）
```
