# ノウハウ記事 定期更新フロー

## 起動方法

`/knowhow-update` を起動するだけ。あとはClaudeが判断する。

---

## Claudeの実行フロー

### STEP 1：今週の対象グループを判定する

今日の日付からISO週番号を計算する。
- 奇数週（週番号が1・3・5…）→ グループ「奇数週」の記事が対象
- 偶数週（週番号が2・4・6…）→ グループ「偶数週」の記事が対象

---

### STEP 2：対象記事を選ぶ

まず何本対応するかを確認する。

```
今日は何本対応しますか？
```

回答を受けてからMCPで維持記事管理シート（ID：`1yr1mz3nQmk2YW2qkjlP0wqZyzdK5ReECz7TdsD9KJfw`）の「ノウハウ一覧」タブを読み込む。

**起動時チェック**：I列に`作業中`が残っている記事があれば、以下を通知してから止まる。
```
前回のセッションで作業中のまま止まっている記事があります。

[記事名]（I列：作業中）

対応してください：
・「完了」→ 今日の日付をI列に書きます
・「リセット」→ 作業中を解除して次の記事に進みます
```

**記事の選定**：対象グループの中から、以下の条件で**I列（更新完了日）が古い順に指定本数**を選ぶ。
- I列が`作業中`の記事は除外する（他インターンが対応中）
- 更新完了日が空欄の記事を最優先とする
- 同じ日付が複数ある場合は行番号が上のものを優先

以下の形式でインターンに提示する。確認なしに次へ進まない。

```
今日の対象記事（[奇数週 or 偶数週]・[競合分析 or SERPチェック]）

1. [記事名]（[サイト]）
   メインKW：[メインKW]
   URL：[URL]
   前回更新：[更新完了日 or 未更新]

2. [記事名]（[サイト]）
   …

3. [記事名]（[サイト]）
   …

この[N]記事を順番に進めましょう。1記事ずつ対応します。
```

**確認後すぐに仮ロックを書き込む**：インターンが「はい」と答えたら、STEP 3に進む前に3記事のI列に`作業中`をMCPで書き込む。これにより他インターンが同じ記事を選ばなくなる。

---

### STEP 3（競合分析のとき）：競合構成を自動取得する

インターンの手動作業は不要。Claudeが以下を実行する。

1. 対象記事のメインKWでWebSearchを実行（自社ドメインを除外）
2. 上位3件のURLをWebFetchでH2/H3見出しを取得
3. 自社記事のURLもWebFetchで構成を取得
4. ギャップ分析結果を出力する

---

### STEP 3（SERPチェックのとき）：インターンにSERPデータを依頼する

```
以下をGoogleで検索して、チャットに貼り付けてください。

メインKW：「[メインKW]」

① AI概要（画面上部に表示される場合）：全文をコピー。表示なしなら「表示なし」
② 他の人はこちらも質問（PAA）：表示されている質問を上から3つ
③ 関連する検索：画面最下部の一覧をすべて
```

貼り付けを受けてからギャップ分析に進む。

---

### STEP 4：不足トピックの候補を出力し、1つだけ選んでもらう

競合分析またはSERPチェックの結果をもとに、自社記事に不足しているトピックを列挙する。
判定基準：記事本文でそのトピックに実質的に回答されているか。見出しがなくても本文で回答済みなら候補にしない。

```
【不足トピック候補】
① [トピック名]：[なぜ入れると良いか1文]
② [トピック名]：[なぜ入れると良いか1文]
③ [トピック名]：[なぜ入れると良いか1文]

1つだけ選んでください。次回以降のセッションで残りを順番に追加します。
```

インターンの返答を受けてから次へ進む。

---

### STEP 5：記事のHTMLを受け取る

```
対象記事のHTMLをWordPressの「テキスト」タブから全文コピーして貼り付けてください。
```

受け取ったHTMLから使用タグ・class名・文体を読み取る。

---

### STEP 6：HTMLを出力する

受け取ったHTMLのタグ構造・class名・文体に合わせて差分HTMLを出力する。
WPの「テキスト」タブに貼り付けて実装できる状態で出す。

実施前に以下を読む。
- `knowledge/block-rules.md`
- `knowledge/guidelines.md`（2章：見出し構成のみ）

---

### STEP 7：スプシを更新する（記事ごとに即時実行）

WP実装が完了した記事ごとに、その場でMCPで以下を書き込む。

**ノウハウ一覧タブ（対象行）**
- I列（更新完了日）：今日の日付（YYYY-MM-DD）← `作業中`を実際の日付で上書きする
- H列（次回タイプ）：現在が「競合分析」なら「SERPチェック」に、「SERPチェック」なら「競合分析」に切り替える
- J列（改修メモ）：追加・強化したH3と理由を1〜2文で記録する

**更新ログタブに1行append**

タブ名：`更新ログ-YYYY-MM`（今日の日付から生成。例：`更新ログ-2026-05`）。タブが存在しない場合は作成してからヘッダー（更新日・サイト名・記事分類・メインKW・更新タイプ・更新内容）を1行目に書き込む。

| 列 | 値 |
|----|-----|
| A（更新日） | 今日の日付（YYYY-MM-DD） |
| B（サイト名） | ノウハウ一覧タブのA列（サイト名） |
| C（記事分類） | ノウハウ |
| D（メインKW） | ノウハウ一覧タブのD列（メインKW） |
| E（更新タイプ） | 競合分析 / SERPチェック |
| F（更新内容） | J列に書いた改修メモと同じ内容 |

次の記事に進む前に必ずSTEP 7を完了させる。
