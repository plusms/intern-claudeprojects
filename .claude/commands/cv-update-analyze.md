# /cv-update-analyze

## STEP 1：今週のタイプを判定する

ISO週番号を計算する。
- 奇数週 → SERP分析
- 偶数週 → ヒートマップ分析

今週のタイプをインターンに提示する。確認なしに次へ進まない。

---

## STEP 2：対象サイトを確認する

以下から対象サイトを選んでもらう。

```
どのサイトのCV記事を更新しますか？
honbe / gyn / asami / ginza / smc
```

---

## STEP 3：対象記事を取得する

維持記事管理シート（ID：`1yr1mz3nQmk2YW2qkjlP0wqZyzdK5ReECz7TdsD9KJfw`）の `[サイト]-CV` タブを読み込む。

**SERP分析週の対象条件**：F〜K列が入力済み、かつN列が空
**ヒートマップ週の対象条件**：O列が空

M列（更新完了日）が古い順に1件を選ぶ。空欄を最優先とする。

以下の形式でインターンに提示する。確認を得てから次へ進む。

```
今日の対象記事（[SERP分析 or ヒートマップ分析]）

記事名：[記事名]
URL：[URL]
メインKW：[メインKW]
前回更新：[更新完了日 or 未更新]

この記事を進めましょう。
```

---

## STEP 4A（SERP分析週）：改修方針案を生成する

1. F〜K列からAIO・PAA・関連KWを読み込む（メインKW・サブKW両方）
2. L列の競合URLをWebFetchでH2/H3を取得（最大3件）
3. 自社記事URLもWebFetchで構成を取得
4. ギャップ分析 → 自社記事に不足している改修方針案を番号付きで生成する

```
【SERP改修方針案】
① [方針]：[理由1文]
② [方針]：[理由1文]
③ [方針]：[理由1文]
…
```

書込前チェック → [N列: SERP改修方針案] | [AIO/PAA/競合分析から生成] | [関連セル矛盾なし?]

承認を得てからN列に書き込む。

---

## STEP 4B（ヒートマップ週）：ヒートマップを分析する

以下をインターンに依頼する。

```
Clarityから以下のスクショを撮影してチャットに貼り付けてください。

対象記事URL：[URL]
① クリックヒートマップ
② スクロールヒートマップ
③ アテンションヒートマップ
```

スクショを受け取ってから分析 → 改修方針案を番号付きで生成する。

```
【ヒートマップ改修方針案】
① [方針]：[理由1文]
② [方針]：[理由1文]
…
```

書込前チェック → [O列: ヒートマップ改修方針案] | [スクショ分析結果] | [関連セル矛盾なし?]

承認を得てからO列に書き込む。

---

## STEP 5：Slack通知テキストを出力する

以下をそのままコピーして社員にSlackで送る。

```
[記事名]の分析が完了しました。
スプシのP列に採用番号を記入してください。
```
