# /cv-update-urls

## STEP 1：対象サイトを確認する

以下から対象サイトを選んでもらう。

```
どのサイトのCV記事を確認しますか？
honbe / gyn / asami / ginza / smc
```

---

## STEP 2：スプシを読み込む

維持記事管理シート（ID：`1yr1mz3nQmk2YW2qkjlP0wqZyzdK5ReECz7TdsD9KJfw`）の `[サイト]-CV` タブを読み込む。

全記事の以下を取得する。
- A列（記事名）
- C列（メインKW）
- L列（前回の競合URL）

---

## STEP 3：競合URLを取得・仕分けする

全記事のメインKWでWebSearchを並列実行し、上位5URLを取得する。

各URLを以下の基準で仕分ける。

| 種別 | 判断基準 |
|------|---------|
| 公式クリニック | クリニック名ドメイン・/access・/staff等の院情報ページ構成 |
| アフィリエイトメディア | 比較・おすすめ系スラッグ・メディア名ドメイン |
| 判断困難 | どちらとも言えない場合 |

前回のL列と比較して差分を確認する。

以下の形式で一覧を提示する。確認なしに次へ進まない。

```
【競合URL確認（[サイト名]）】

■ [記事名]（[メインKW]）
  1位：[URL]（アフィリ）
  2位：[URL]（公式）
  3位：[URL]（アフィリ）
  4位：[URL]（公式）
  5位：[URL]（アフィリ）
  差分：新規 [URL]（アフィリ）★ / 変化なし

■ [記事名2]（[メインKW]）
  ...

仕分けに間違いがあれば修正してください。
```

**新規アフィリエイトメディアが出現した記事**はWebFetchでH2/H3を取得し、差分サマリーを合わせて提示する。

```
【新規競合 重点分析：[URL]】
競合の主要見出し：[H2/H3]
自社記事との差分：[不足コンテンツ]
対応要否：[要対応 / 経過観察]
```

---

## STEP 4：L列に書き込む

書込前チェック → [L列: 競合URL] | [WebSearch取得結果] | [関連セル矛盾なし?]

承認を得てから全記事分まとめてL列に書き込む（上書き）。

---

## STEP 5：ログをmdに追記する

`logs/[サイト名]-cv-log.md` に以下の形式で追記する。ファイルが存在しない場合は新規作成する。

```markdown
## [YYYY-MM-DD] 競合URLチェック

### [記事名]（[メインKW]）
1位：[URL]（アフィリ）2位：[URL]（公式）…
差分：新規 [URL]（アフィリ）★ / 変化なし

---
```
