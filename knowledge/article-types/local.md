# local.md｜地域記事の前提ルール

CV施策・順位施策を問わず、地域記事を扱う際に必ず読む。

---

## 地域記事とは
メインKWに地域名（都市名・エリア名・駅名等）が含まれる記事。
例：「クマ取り 札幌」「シミ取り 札幌」「ICL 梅田」

---

## 絶対ルール

### 1. 地域名置換テスト
**すべてのH3本文に「地域名を別地域に置換しても成立しない文」が1文以上あること。**
- NG：「〇〇でシミ取りを受ける際は、料金だけで選ばないことが大切です」
- OK：「札幌駅・大通周辺には大手チェーンが集中しており、複数院を同日に比較しやすい環境が整っています」

### 2. サブエリアの粒度
都市名で一括りにしない。主要サブエリア（駅・エリア名）の粒度まで書く。
- NG：「札幌のクリニックは〜」
- OK：「札幌駅周辺・大通周辺・すすきの周辺で施術傾向が異なる」

### 3. 地域固有情報は削らない
地域名・エリア名・地域の料金傾向・地域ユーザーの通院事情は削除・縮小しない。
これらを削ると「他のKWでも言える記事」になり、地域KWでの順位を失うリスクがある。

### 4. 地域断言には根拠が必要
「〇〇地域は〜な傾向がある」という記述には必ず根拠を示す。
- NG：「札幌は料金が安い傾向があります」（根拠なし）
- OK：「調査した10院の平均費用は約190,440円でした」（実調査ベース）

### 5. 地域ユーザー文脈を入れる
その地域のユーザーが抱えやすい動機・不安・移動前提を反映する。
- 例（札幌）：冬場の雪・外出負担・屋内通路での来院可否
- 例（地方都市）：車移動前提・駐車場の有無が重要
- 医療ジャンルは「通いやすさ」より「実績・専門性・費用透明性」が主軸

### 6. クリニック・施設紹介文の地域性
各クリニック紹介文が「名称・住所・特徴の羅列」になっていないか確認する。
- 他地域でも言えるブランド全体の実績・全国共通の料金は地域固有情報に差し替える
- 「なぜその施設を札幌で選ぶか」が伝わる紹介文にする

## 順位施策でのフロー全体像

```
rank-sc-check（GSCクエリ分析）
        ↓
rank-local-check
（6軸チェック＋競合地域差分確認）
        ↓
  対応リストのルーティング判断
        ↓
  ┌─────────────────────────────┬──────────────────────────────────┐
  │ 構造問題あり                 │ 品質問題のみ                      │
  │ ・新H2/H3の追加が必要        │ ・構造は維持                      │
  │ ・H2の削除・統合・順番変更   │ ・既存セクションの深掘り・         │
  │ ・論点化セクション自体がない │   PREP修正・情報補完のみ           │
  ↓                             ↓
rank-reform-design          rank-edit-html（直接）
  ↓
rank-edit-html
```

**判断に迷ったら：** 対応リストに「新H2/H3を追加する」「H2を削除・統合する」が1件でもあれば rank-reform-design を挟む。
